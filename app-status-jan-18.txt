Product Status Report - January 18, 2026
================================================================================

PRODUCT OVERVIEW
================================================================================

Barrio is a hyperlocal events discovery iOS application that enables users to
discover and create local events within a 5-kilometer radius. The app focuses
on simplicity and local community engagement, allowing users to share events
with photos or videos, view nearby events on an interactive map, and interact
through Likes and Going indicators.

Product Philosophy:
- Single user type (no business accounts)
- No social graph (no following/followers)
- No comments, stories, or saved posts
- Focus on local, time-bound events
- Map-first discovery experience

Target Platform:
- iOS 17+ (iPhone only)
- Portrait orientation only
- TestFlight-ready MVP

- LocationManager (ObservableObject) - Location services
  * CLLocationManager wrapper

- StorageService (actor) - Media uploads

  * Video duration validation
  * Image compression

Views Layer:

- ViewModels for complex state management
- Environment objects for dependency injection

Key Design Decisions:
1. Actor-based services for thread safety

MEDIA UPLOAD ARCHITECTURE
-------------------------
The app uses a hybrid upload approach:

1. Direct Client Uploads (Primary Method):
   - iOS requests signed URL from server
   - Server validates (duration, type) and generates Supabase signed URL
   - iOS uploads directly to Supabase Storage
   - Much faster (bypasses server bandwidth)
   - Better scalability

2. Server Proxy Upload (Fallback):
   - Legacy endpoint for compatibility
   - Base64-encoded media sent to server
   - Server decodes and uploads to Supabase
   - Slower but more controlled

Benefits:
- Reduced server load
- Faster uploads (direct to Supabase)
- Better user experience
- Scalable architecture

GEOSPATIAL QUERIES
------------------
PostGIS Integration:
- Location stored as PostGIS geometry (geography type)
- GIST index for efficient spatial queries
- ST_DWithin for 5km radius filtering
- ST_Distance for distance calculations
- Automatic geometry updates via database triggers

Query Pattern:
```sql
SELECT * FROM events
WHERE ST_DWithin(
  location::geography,
  ST_SetSRID(ST_MakePoint(lng, lat), 4326)::geography,
  5000  -- 5km in meters
)
AND (endTime > NOW() OR (endTime IS NULL AND startTime > NOW()))
ORDER BY created_at DESC
LIMIT 50
```

Performance:
- GIST index enables fast spatial queries
- Sub-100ms query times for nearby events
- Efficient distance calculations
